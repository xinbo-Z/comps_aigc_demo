# Table 列宽拖拽与持久化

## 能力概述

`columnResize` 用于开启列宽拖拽，`onColumnsChange` 会在拖拽结束后返回整份更新后的 `columns`。Table 内部不持久化列宽，最终列宽由外部保存并重新传回。

## 最小受控示例

```tsx
import { useState } from 'react'
import { Table, type TableColumnsType } from '@sci-comp/core'

interface TableRow {
  key: string
  name: string
  owner: string
}

const data: TableRow[] = [
  { key: '1', name: '名称', owner: '张三' },
  { key: '2', name: '负责人', owner: '李四' },
]

const initialColumns: TableColumnsType<TableRow> = [
  { title: '名称', dataIndex: 'name', key: 'name', width: 180 },
  { title: '负责人', dataIndex: 'owner', key: 'owner', width: 140 },
]

export function Demo() {
  const [columns, setColumns] = useState(initialColumns)

  return (
    <Table
      rowKey="key"
      columns={columns}
      dataSource={data}
      columnResize
      onColumnsChange={setColumns}
    />
  )
}
```

## 持久化落点

- 页面内临时状态：React state
- 单端恢复：localStorage
- 多端一致：服务端用户偏好

组件已经把完整的 `nextColumns` 提供给业务，业务只需要保存并回传，不需要再从事件里手动拼装列宽差异。

## 使用约束

- 需要稳定列标识，优先使用 `key`，其次支持单字段 `dataIndex`
- grouped columns 不参与拖拽
- 拖拽只修改当前列宽
- 宽表建议配合 `scroll.x`
- `minWidth` 用于保护最小列宽，不负责完整布局系统
