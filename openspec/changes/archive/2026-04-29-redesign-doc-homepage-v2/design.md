## Context

当前 `sci-comp-documention` 首页已经完成过一轮信息架构层面的重组，但实际落地效果仍然偏克制、偏说明型，距离用户期待的“炫丽”“好看”“有设计感”的首页还有明显差距。问题不在于区块数量不够，而在于视觉语言仍过于保守：Hero 的首屏冲击力不够，组件展示区还不够像产品级拼贴，卡片体系缺少足够强的层次变化，整页节奏也没有形成成熟热门组件库首页那种“首屏惊艳 + 分区耐看”的完成度。

本次调整不是回退到营销官网，也不是改造整站主题，而是在保留文档导航语义、中文表达与现有技术边界的前提下，把首页重新设计成更具视觉表现力的组件库首页。实现约束仍然存在：不能引入复杂动画系统、额外视觉依赖或脱离项目真实能力的夸张宣传，但可以大胆使用渐变、光效、层叠容器、悬浮感卡片、分区底色和更强的排版对比来提升质感。

## Goals / Non-Goals

**Goals:**

- 将首页升级为具有更强视觉吸引力、展示感和品牌识别度的炫丽型组件库首页
- 保留首页作为文档入口页的职责，同时显著提升首屏完成度、卡片观感和整体版式节奏
- 让 Hero 展示区真正体现组件体系的产品化观感，而不是简单 mock 信息拼装
- 通过更丰富的分区背景、卡片层次和内容编排，让首页整体接近热门组件库文档站的首页设计完成度
- 在不引入额外依赖的前提下，利用现有 token 与 TSX 内联样式实现可维护的高质感首页

**Non-Goals:**

- 不改造全站导航、侧边栏或已有组件详情页内容结构
- 不引入复杂动效库、3D 引擎、额外第三方视觉框架或在线编辑器
- 不将首页改造成与文档站脱节的纯营销官网
- 不夸大组件库实际能力边界，不通过虚假能力文案制造视觉承诺
- 不扩大为全站主题系统重构

## Decisions

### 1. 首页风格从“克制企业级”升级为“炫丽但仍可信”的组件库首页

整体方向从上一版的保守企业级入口页，调整为更强视觉表现力的首页：允许更大胆的渐变背景、光效、描边、发光容器和层叠卡片，但仍保持中文文档站的可信感与导航价值。

- 选择该方向的原因：用户明确反馈当前效果“太过简陋”，说明仅靠信息架构升级无法满足预期，必须进一步提高视觉完成度。
- 替代方案：继续在现有方案上做小幅样式润色。
- 不选原因：只能得到“更整齐的目录页”，很难变成用户期待的炫丽首页。

### 2. Hero 采用“大背景氛围 + 多层展示拼贴 + 强 CTA”结构

首屏需要成为整页最强视觉中心：左侧保留定位文案与 CTA，右侧升级为更完整的组件系统展示舞台，并加入背景光晕、浮层、状态条、数据卡、表单卡和辅助统计信息。

- 选择该结构的原因：热门组件库首页的第一印象通常来自 Hero 的整体舞台感，而不是单纯的标题和按钮。
- 替代方案：保留双栏结构，只微调右侧 card。
- 不选原因：无法显著改变首页的第一视觉印象。
- 展示区至少应体现按钮、数据卡、表格或进度、表单或 SchemaForm 等 3 类以上组件语义，并形成前中后景关系，而不是平铺几张同级小卡片。

### 3. 各信息区块使用差异化容器系统，而不是统一普通卡片网格

首页不能再让所有分区都使用同一类浅色卡片。不同区块需要有更明显的背景差异、容器密度和视觉节奏，例如深浅交替分区、强调型大卡、次级信息侧栏、数据标签和分层投影。

- 选择该方式的原因：当前“简陋感”的主要来源之一，就是所有信息都在同一视觉权重下呈现。
- 替代方案：继续统一 card 样式，只靠文案区分。
- 不选原因：文案再丰富，也无法弥补视觉节奏单一的问题。

### 4. 核心组件与高阶能力要兼具入口性与展示感

Featured Components、More Components 与 SchemaForm 高阶能力区仍然保留，但样式上不再只是普通链接卡，而要具备更强的封面感、标签感、局部预览感或结构化亮点信息，让它们更像热门组件库首页中的“能力入口模块”。

- 选择该方式的原因：用户要求首页“样式好看”，组件入口区必须同步提升，否则只会出现“Hero 好看，下面还是普通文档卡片”的割裂感。
- 替代方案：只强化 Hero，其他区块保持原样。
- 不选原因：整页视觉质量会断层。

### 5. 实现仍基于现有 token 与 TSX 内联样式，但允许更复杂的局部展示组件

实现层面继续沿用 `@sci-comp/core` token 和 `HomePage.tsx` 内联样式，不新增样式基础设施；但允许把首页拆出更多局部函数组件，如展示卡、统计条、浮层标签、预览面板等，以支撑更高完成度的视觉结构。

- 选择该方式的原因：既能控制实现范围，又能满足更高完成度的首页需求。
- 替代方案：引入新的全局样式层或动画库。
- 不选原因：会扩大复杂度，降低可维护性。

### 6. 新版首页的成功标准

首页改造完成后，应至少满足以下判断标准：

- 用户首屏第一眼会感受到明显高于普通文档目录页的视觉完成度和展示感
- 首页 Hero 既具备炫丽感，又不丢失快速开始、组件浏览和高阶能力入口
- 主展示区能清晰表达“基础组件 + 数据展示 + 高阶表单能力”并存的体系认知
- Featured、More Components 与 SchemaForm 区块在视觉上不再只是统一平面卡片，而具备更强模块化和封面感
- 整页具备明确的节奏变化和分区层次，而不是从上到下都像同一种信息面板
- 新设计仍可在现有文档工程中维护，不依赖重型视觉系统

## Risks / Trade-offs

- [视觉表达更强可能压过导航语义] → 保证 CTA、入口卡和路径区仍然具有明确链接与阅读指向
- [首页过于炫丽可能与项目真实能力不匹配] → 所有展示模块都基于现有组件语义拼装，不使用脱离实际能力的虚构产品图
- [样式复杂度升高后维护成本上升] → 限制在 `HomePage.tsx` 局部组件与 token 体系内，不扩展为新基础设施
- [移动端容易出现拥挤] → 在布局上采用响应式堆叠，并控制首屏展示模块的最小宽度与层级数量
