## Context

当前首页已经历过两轮方向演进：第一轮从普通目录页升级为企业级组件体系入口页，第二轮又进一步强化为炫丽型展示页。但现在新的参考对象变成了 shadcn 中文站首页，这意味着方向再次发生变化：目标不再是“更炫”，而是“更高级、更克制、更像成熟设计系统入口页”。

shadcn 风格的关键不在于大面积视觉特效，而在于留白、排版秩序、轻边框、低装饰密度和单一主舞台式的产品预览。对于当前项目来说，这种方向并不是要抹掉首页的组件体系感，而是要把这种体系感从“多卡片强展示”转为“更少模块、更稳结构、更像真实产品工作区”的表达方式。项目本身仍然是基于 Ant Design v6 的业务组件文档站，因此不能照搬 shadcn 的代码片段站语境，但可以借鉴它的视觉组织方式。

## Goals / Non-Goals

**Goals:**

- 将首页从炫丽型展示页收敛为更接近 shadcn 风格的极简产品化首页
- 保留首页作为文档入口页的职责，同时显著降低装饰密度，提升排版秩序与留白质量
- 让 Hero 右侧展示区更像单一产品工作台，而不是多个并列展示卡拼贴
- 收敛首页中段 section 数量和视觉容器层级，形成更少但更稳的主模块
- 用更克制的颜色和边框系统建立高级感，而不是继续依赖重渐变、发光和重阴影

**Non-Goals:**

- 不直接复刻 shadcn 的品牌视觉、文案语气或网站结构
- 不把首页改造成代码片段市场、组件安装导航站或纯开源官网
- 不改变当前文档站的中文表达、组件体系定位和主路径入口职责
- 不扩展为整站主题系统重构或组件详情页重做
- 不删除 SchemaForm 作为高阶能力亮点的首页地位

## Decisions

### 1. 整体方向从“炫丽展示”切换为“极简产品化”

首页整体视觉语言向 shadcn 风格靠近：减少大面积渐变、发光和多层装饰，转而使用更大的留白、更稳的栅格、轻边框和更克制的强调色。

- 选择该方向的原因：用户点名参考 shadcn 中文站，说明当前问题不是“还不够炫”，而是首页仍然不够高级、不够克制。
- 替代方案：在现有炫丽型首页上继续做微调。
- 不选原因：只会得到“更收敛的展示页”，不会真正切到 shadcn 式设计系统入口风格。

### 2. Hero 采用“大标题 + 短副标题 + 单一工作台式预览”

Hero 左侧保留主标题、短副标题和最少必要 CTA，右侧则从多块拼贴收敛为单一主舞台，用更像真实工作区的结构去同时表达交互、数据展示和 SchemaForm 能力。

- 选择该结构的原因：shadcn 风格的高级感来自焦点集中，而不是信息并列堆叠。
- 替代方案：继续使用多个展示卡拼贴，只是视觉做轻量化。
- 不选原因：仍然会保留“展示面太满”的问题。

### 3. 将信任信号与指标信息从 Hero 主叙事中弱化

当前首页首屏中包含较多信任信号和指标卡。新方向下，这些信息应当从主叙事中降级，转为 Hero 下方的轻量信息带、次级标签行或更简短的数据摘要，而不是继续占据首屏主要视觉重心。

- 选择该方式的原因：shadcn 风格更依赖标题与空间，而不是大量首屏辅助卡片。
- 替代方案：保留 heroMetrics / trustSignals 的卡片结构。
- 不选原因：会破坏 Hero 的留白和聚焦感。

### 4. 中段区块收敛为更少的主模块

当前首页包含 Why、Pathways、Capabilities、Featured、Advanced Highlight、Quick Start 等多个 section。新方向下应收敛为更少但更稳的模块，例如：Why/Capabilities 合并、Pathways 独立、Featured + SchemaForm 形成主能力区、Quick Start 保留收尾。

- 选择该方式的原因：模块越多越容易回到 landing page 堆叠感，shadcn 风格更强调少而稳。
- 替代方案：保留所有现有 section，仅降低装饰强度。
- 不选原因：结构节奏仍然过碎。

### 5. 卡片系统优先强调秩序，而非效果

路径卡、能力卡和组件入口卡不再追求强封面感、重阴影和大面积渐变，而应依赖更统一的边框、标题层级、标签与留白建立秩序感。强调色只在极少数关键位置出现。

- 选择该方式的原因：shadcn 风格的高级感更多来自设计秩序，而不是效果叠加。
- 替代方案：维持当前 feature card 的展示感，只缩小强度。
- 不选原因：仍然会显得太“做设计”。

## Risks / Trade-offs

- [过度收敛后首页可能显得不够有记忆点] → 保留单一高质量 Hero 主舞台与少量强调色，避免完全素净到失焦
- [参考 shadcn 可能削弱项目自身业务组件定位] → 保留 SchemaForm、高阶能力与文档入口职责作为首页核心叙事
- [模块收敛后可能丢失部分现有信息] → 通过合并区块而不是删除关键信息来控制范围
- [实现时容易变成“把所有颜色都去掉”] → 明确目标是高级感和产品感，不是低配极简
